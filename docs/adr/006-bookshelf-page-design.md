# ADR-006: 書櫃頁設計決策

- 狀態：已接受（設計定稿）
- 日期：2026-07-22
- 相關：ADR-005、`docs/spec.md` §3.1 `books`、§10 前端對應
- 備註：分類（genre）欄位為獨立的資料模型決策，另見 ADR-007（討論中）。


---

## 背景
原「書櫃」路由實際上是單本書的詳情頁（detail）。新增一個真正的書櫃列表頁 `passbook.app/bookshelf`，作為 §10 提到的「書架（有封面）」檢視的完整實作。設計經過多輪迭代收斂，過程中浮現一個會阻擋切版的 schema 缺口： `books` 表沒有分類欄位，但書櫃頁的分區、篩選，以及統計頁的類型分布都依賴它。


---

## 決策：書櫃頁採雙檢視（層板 / 書背）

提供兩種檢視，透過頁首 chip 切換：
| 檢視 | 定位 | 互動 | 呈現 |
| --  |  --  | --  |  -- |
| 層板 | 資訊與細讀 | 直向網格捲動 | 封面正面朝外，下方顯示書名 + 進度兩行 |
| 書背 | 氛圍與快速瀏覽 | 手機為列內橫向捲動(slider) | 書背直立並排，直排書名，資訊極簡 |


理由：兩個檢視的價值來自「一格橫掃、一個直讀」的分工。若兩者都做成橫向 slider 或都做成資訊網格，功能就重疊、其中一個失去存在理由。切版時需維持此差異，不要讓層板檢視也變成橫向捲動。


### 從屬的視覺決策（皆為最終樣式）

- 層板等長、無書擋：層板寬度統一貼齊內容區，書靠左排，右側為空層板。空層板本身即傳達「還有位子」，不需要書擋。（迭代過程中的細桿書擋造型失敗，已移除。）
- 層板厚度：桌機與手機一致，6px 實色 + 下方陰影，不可在手機退化成細分隔線。
- 傾斜書（斜書）：每個類型分組的最後一本以底部左角為軸傾斜約 10-12 度，倚靠前一本書，作為該列結尾的視覺收束，呼應側邊欄書櫃 icon（三條直線與一條向左斜線來示意書櫃）。書背檢視因此不需書擋。
- 閱讀中狀態：以書頂垂下的金色書籤絲帶（#E0B654）表示，不用橫帶蓋住封面。
- 書背厚度對應頁數：高度固定，以寬度對應頁數（語意上書的厚度才是頁數）。


### 切版注意事項

- 斜書是純展示層：僅為 CSS `transform: rotate()`，`transform-origin: bottom left`。傾斜與否只由「是不是該類型分組的最後一本」決定，不進資料模型。
- 斜書的 RWD 邊界需實測：旋轉後的觸控命中區、相鄰書的 z-index (斜書需在最上層)、橫向捲動容器的 `padding` 要留夠以免斜書被 `overflow` 切掉。
- 該列只有 1 本書時不傾斜。
- 所有可點擊區塊觸控範圍 ≥ 44x44px。

## 曾嘗試但放棄的做法

紀錄迭代過程中考慮或實作過、但最終放棄的方向，說明放棄理由，避免日後重蹈覆轍。

| 放棄的做法 | 放棄理由 |
| -------- | ------ |
| hover 才顯示書名/進度 | 桌機思維，mobile 沒有 hover 效果，會直接失去資訊。改為層板檢視書名常駐、書背檢視書名直排印在書背上。 |
| 書平躺水平長條推疊當「書背」| 方向錯誤——那是書平放疊高，不是書櫃。書背的美感來自於「直立、窄、並排」。改為書背垂直站立、水平並排、可橫向捲動。|
| 細桿書擋 | 造型比例過細過高像點滴架，且離最後一本書太遠、沒有實際支撐感。最終直接移除——等長空層板本身已傳達「還有位子」。 |
| 層板寬度隨書的數量縮短 | 導致各列右緣參差不齊，視覺感受雜亂。改為所有層板等長，書靠左排、右側留空層板，維持整齊感。 |
| 層板檢視在手機也做橫向 slider | 會與書背檢視功能重疊，且橫向捲動讓使用者「掃過」而非「讀」書名與進度，違背層板檢視「細讀」的定位。層板檢視維持直向網格捲動 |
